## 🔄 2. 任務隊列管理 (Queue Management Optimization)

### A. 由遞步轉為並行 (DAG 並行調度)
目前系統採用線性執行的隊列 (Sequential Queue)。針對高效能知識系統，應優化為 **DAG（有向無環圖）** 分發機制：
*   **同步發動**：A1 (數據採集) 與 A4 (歷史檢索) 應同時執行，不需互相等待。
*   **階段性屏障 (Barriers)**：確保所有基礎採集任務結束後，再交由 A2 進行決策建模。

### B. 優先級與動態分層 (Priority Tiers)
*   **第一級 (Interactive)**：使用者即時問答，擁有絕對優先執行權。
*   **第二級 (Backfill)**：後台自動化的知識庫掃描與節點擴展。

## 🧠 3. 單一代理人迭代 (Focus Mode) 與破解「偷懶原則」

### 克服模型的偷懶本性 (Overcoming Lazy Refinement)
大語言模型在連續輪詢中容易產生「確認偏誤 (Confirmation Bias)」，傾向於重複先前的答案。為了榨出單個 Agent 的最大價值，必須引入 **對抗式迭代 (Adversarial Iteration)**：

1.  **第一輪 [初稿建立]**：生成基礎邏輯架構。
2.  **第二輪 [嚴苛審計]**：**強制轉換視角**。
    *   *戰術指令*：「請扮演一名極度挑剔的審計員。找出你在第一輪回答中的 3 個邏輯漏洞、假設錯誤、或過於空泛的點，並顯式列出。**禁止**讚美或同意之前的任何觀點。」
3.  **第三輪 [整合修正]**：**最終收斂**。
    *   *戰術指令*：「結合第一輪的草稿與第二輪發現的具體漏洞，執行最終的重構。確保所有已識別的缺點在這一輪中都得到解決。」

## 🛠️ 4. 針對「知識節點」系統的特定優化

### A. 脈絡快取 (Context Caching)
針對同一個龐大知識庫進行連續問答時，必須啟用脈絡快取。
*   **效益**：這在 2026 年標準中能降低 60% 以上的 API 延遲與長文本預處理成本。

### B. 推理鏈視覺化 (Reasoning-Trace)
不要隱藏 Agent 的思考過程。在 Pathway 視圖中，應將模型的 `thought` (思考鏈) 渲染為隱性節點，方便用戶進行邏輯溯源與除錯。

### C. 節點深度下鑽 (Drill Down)
當用戶手動點擊某個 Agent 節點時，系統應：
*   自動進入單人聚焦模式。
*   自動引用該節點之父節點 (Parent Node) 的數據作為任務上下文。
